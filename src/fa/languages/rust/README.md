# دستورالعمل‌های مهندسی Rust

این بخش استانداردها و بهترین روش‌ها را برای توسعه Rust در Dita شرح می‌دهد. هدف ما بهره‌گیری از تضمین‌های ایمنی Rust در عین حال حفظ کدی خوانا و قابل نگهداری است.

## اصول اصلی

1.  **ایمنی در اولویت (Safety First)**: از کد `unsafe` اجتناب کنید مگر اینکه کاملاً ضروری باشد و به شدت ایزوله و مستند شده باشد.
2.  **راست اصولی (Idiomatic Rust)**: از قراردادهای استاندارد پیروی کنید. اگر در حال «جنگیدن با borrow checker» هستید، طراحی را بازنگری کنید.
3.  **مستندسازی (Documentation)**: تمام APIهای عمومی باید مستند شوند.

## ابزارها

*   **فرمت‌کننده (Formatter)**: ما از `rustfmt` با پیکربندی استاندارد استفاده می‌کنیم.
*   **لینتر (Linter)**: ما از `clippy` استفاده می‌کنیم. پایپ‌لاین‌های CI باید در صورت وجود هشدارهای clippy با خطا مواجه شوند.
