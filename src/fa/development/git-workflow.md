# جریان کاری گیت

این بخش جریان کاری گیت ما را که شامل شاخه‌گذاری، کامیت کردن و مدیریت موضوعات (issues) است، مشخص می‌کند.

1.  **شاخه‌گذاری**: برای هر وظیفه یک شاخه جدید بسازید. به [استراتژی شاخه‌گذاری](./branching-strategy.md) مراجعه کنید.
2.  **کامیت کردن**: پیام‌های کامیت واضح و طبق قرارداد بنویسید. به [راهنمای کامیت](./commit-guidelines.md) مراجعه کنید.
3.  **ایخوها (Issues)**: کار خود را به ایشوهای مشخص لینک کنید. به [مدیریت ایشوها](./issue-management.md) مراجعه کنید.
4.  **ADRها**: در صورت لزوم به تصمیمات معماری ارجاع دهید. به [لینک‌دهی ADR](./adr-linking.md) مراجعه کنید.

## مراحل جریان کاری

1.  یک ایشو انتخاب کنید (یا یکی بسازید).
2.  یک شاخه از `dev` (برای ویژگی‌های جدید) یا `master` (برای رفع حوادث) بسازید.
3.  تغییرات را پیاده‌سازی کنید.
4.  تغییرات را با پیروی از [راهنمای کامیت](./commit-guidelines.md) کامیت کنید.
5.  شاخه را پوش کنید و یک Pull Request باز کنید.
6.  اطمینان حاصل کنید که بررسی‌های CI (شامل `commitlint`) پاس شوند.
