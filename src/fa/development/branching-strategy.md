# استراتژی شاخه‌گذاری

ما از استراتژی استفاده می‌کنیم که امکان توسعه موازی با استفاده از شاخه‌های کوتاه-مدت را فراهم می‌کند. مخزن دارای دو شاخه دائمی است: `master` و `dev`.

## شاخه‌های اصلی

-   **master**: وضعیت آماده برای تولید. شامل کد پایدار است.
-   **dev**: شاخه اصلی توسعه. تمام ویژگی‌های جدید ابتدا در اینجا ادغام می‌شوند.

## نام‌گذاری شاخه‌ها و استراتژی

### شاخه‌های ویژگی (Feature Branches)
برای اضافه کردن قابلیت‌های جدید استفاده می‌شود.

-   **شاخه منبع**: `dev`
-   **قرارداد نام‌گذاری**: باید شامل شماره ایشو باشد.
-   **فرمت**: `feat/issue-<number>/<short-description>`
-   **مثال**: `feat/issue-42/add-login-page`

### شاخه‌های رفع باگ (Bug Fix Branches)
برای رفع باگ‌های حیاتی یا حوادث در محیط تولید استفاده می‌شود.

-   **شاخه منبع**: `master` (برای هات‌فیکس‌ها/حوادث ترجیح داده می‌شود)
-   **قرارداد نام‌گذاری**: باید شامل شماره حادثه باشد.
-   **فرمت**: `fix/incident-<number>/<short-description>`
-   **مثال**: `fix/incident-808/resolve-memory-leak`

### سایر شاخه‌ها
برای مستندات، ریفکتورینگ یا کارهای روزمره (chores).

-   **شاخه منبع**: `dev` (معمولاً)
-   **فرمت**: `type/<scope>/<short-description>` یا `type/issue-<number>/<short-description>`
-   **مثال**: `docs/readme/update-setup`

_شاخه‌ها باید پس از ادغام حذف شوند._
